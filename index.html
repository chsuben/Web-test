Design Rationale

Our first idea was to show how different countries' trust changes over time with a world map and a drag bar. However, there were a lot of null values in the dataset, so it's difficult to see the trends. Thus, we choose a world map with countries with different color labels representing the approval rating on Trump's policies regarding certain issues.

The first interaction technique is a drop-down menu. Users can use the drop-down menu to select what issue they are interested in and view the approval rating on the world map. Before we were considering using a drag bar for choosing issues, but drag bar implies continuity while those issues have no such property. We also considered using buttons, but it's hard to tell what those buttons represent. Thus, we use drop-down menu instead of drag bar or buttons.

The second interaction technique is the zooming feature on the world map. Users can click on countries on the map to have a closer view on those countries. Besides that, when the cursor hovers over a country, a text box with the name of the country and the approval rating pops up. We want to make it easier for the users to check the exact approval rating without checking the legend. Users can also have a overview of the rating across the world with the legend.

We wanted to keep the data of change of world's trust over time, so we created another tab view with a map and a line chart. The most common way to see how things change over time is line chart. Thus, we provided a line chart with world's average "trust rating". If a user would like to look into a specific country. He/She can click on the map as before to zoom in, and the trend of that country will also show up in the line chart.
